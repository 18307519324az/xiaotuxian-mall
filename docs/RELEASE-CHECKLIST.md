# 发布检查清单

## 发布硬化说明

当前已进入发布硬化阶段。
当前阶段不新增业务功能，只做：

- 发布前清理
- 公开文档统一
- 发布目录整理
- 最终构建与验证

- [x] 功能范围冻结
- [x] 公共文档安全扫描

## 本轮检查重点

除基础构建外，本轮公开发布还特别关注：

- 仓库定位是否清晰
- 技术栈是否分层准确
- 截图是否存在乱码
- docs 是否足够详细
- GitHub About 与 topics 是否和 README 一致

这些点如果处理不好，公开仓库即使能运行，也仍然不够专业。

## 发布前人工复查建议

在真正对外展示前，建议至少再人工复查以下页面：

- 主仓 README
- Mock README
- 前端 README
- 主仓 docs 入口
- 首页和结算页截图

这样可以避免编码、图片或文案层面的展示问题。

## Contributors 检查

除文档和截图外，公开发布还需要确认：

- `git log --all` 是否只剩目标作者
- 历史 tag 是否仍指向旧提交
- GitHub Contributors 是否只显示目标账号

如果历史作者没有清干净，公开仓库观感依然会受到影响。

## 最终确认顺序

建议最终按以下顺序确认：

1. 本地构建通过
2. 历史作者干净
3. 远端 main 和 tag 正确
4. raw README 内容正确
5. GitHub 页面截图、About、topics 正常

只有这几个步骤同时满足，才能视为公开版真正完成。
